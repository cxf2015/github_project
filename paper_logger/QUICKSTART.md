## 📖 使用流程

### 1️⃣ 添加第一篇论文

#### 方法 A: 从本地 PDF 文件
1. 点击顶部工具栏的 "**📁 打开本地 PDF**" 按钮
2. 选择一个 PDF 文件
3. 输入论文标题 (默认使用文件名)
4. 系统会自动上传文件并创建记录

#### 方法 B: 从 URL
1. 点击顶部工具栏的 "**🔗 打开 URL PDF**" 按钮
2. 输入 PDF 的直接下载链接 (例如 arXiv 论文链接)
3. 输入论文标题
4. 系统会创建记录并加载在线 PDF

#### 方法 C: 通过侧边栏
1. 点击左侧 "**+ 添加论文**" 按钮
2. 填写论文信息
3. 选择本地文件或 URL
4. 点击 "创建论文记录"

### 2️⃣ 阅读和记笔记

1. **选择论文**: 在左侧列表点击论文标题
2. **查看 PDF**: 左侧区域显示 PDF 内容
3. **记录笔记**: 
   - 在右侧底部文本框输入笔记
   - 可选填写页码 (关联到具体页面)
   - 点击 "**💾 保存笔记**"

### 3️⃣ 管理笔记

- **编辑**: 点击笔记右上角的 "✏️ 编辑" 按钮
- **删除**: 点击 "🗑️ 删除" 按钮
- **查看**: 切换不同论文时自动加载对应笔记

### 4️⃣ 数据持久化

所有数据自动保存到:
- `papers.db` - SQLite 数据库 (论文信息和笔记)
- `uploads/` - 上传的本地 PDF 文件

下次打开应用时，所有记录都会自动恢复!

## 💡 使用技巧

### 获取 PDF URL
- **arXiv**: 打开论文页面 → 点击 "PDF" 按钮 → 复制链接
  - 示例：https://arxiv.org/pdf/2301.12345.pdf
- **其他来源**: 确保链接是直接指向 PDF 文件的 (以 .pdf 结尾)

### 笔记组织建议
1. **按章节记录**: 在笔记开头标注章节号
   ```
   [第 3 章] 主要贡献:
   - 提出了新的方法...
   ```
2. **标记疑问**: 使用特殊标记标识需要深入理解的内容
   ```
   ❓ 疑问：这个公式是如何推导的？
   ```
3. **总结要点**: 阅读完整篇论文后添加总结
   ```
   📌 总结:
   创新点：...
   实验结果：...
   启发：...
   ```

### 多设备同步 (高级)
如果需要跨设备使用，可以将整个 `paper_logger` 文件夹放到云盘 (如 OneDrive、Dropbox),这样数据和文件都会自动同步。

## 🔧 常见问题

### Q: 上传的文件存储在哪里？
A: 所有上传的 PDF 文件都存储在 `uploads/` 文件夹中，使用唯一 UUID 命名避免冲突。

### Q: 数据库可以重置吗？
A: 可以。删除 `papers.db` 文件即可重置所有数据 (请谨慎操作)。

### Q: 支持其他文件格式吗？
A: 当前版本仅支持 PDF 格式。如需支持其他格式，可以修改代码扩展。

### Q: 可以在手机上使用吗？
A: 理论上可以，只要手机能访问电脑的 localhost:5000 端口。但界面主要是为桌面端优化的。

### Q: 如何备份数据？
A: 备份整个 `paper_logger` 文件夹即可，包括:
- `papers.db` 数据库文件
- `uploads/` 文件夹中的所有 PDF
